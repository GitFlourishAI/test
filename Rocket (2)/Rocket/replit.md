# replit.md

## Overview

This project is a financial market monitoring and analysis system. It pulls foreign exchange (FX) and other financial data from the Alpha Vantage API, detects anomalies (significant price changes), stores detected events in a PostgreSQL database, and then uses OpenAI-powered AI agents to analyze and reflect on the detected anomalies. The system runs on a scheduled loop, continuously checking for new data, detecting anomalies, saving them, and generating AI-driven insights.

A Palantir-style institutional dashboard (`server.py`) serves as the frontend, displaying catalyst reports generated by the AI agents for family office portfolio monitoring.

## System Architecture

### Data Pipeline Flow

The system follows a linear pipeline architecture:

1. **Data Ingestion** (`alpha.py`) — Fetches financial data (FX daily rates, intraday prices, treasury yields) from the Alpha Vantage API.
2. **Anomaly Detection** (`change.py`) — Compares today's price to yesterday's price and flags significant percentage changes using a configurable threshold (default 1%).
3. **Event Storage** (`signal123.py`) — Saves detected anomalies to a PostgreSQL table (`signal_11`) and provides retrieval of the latest event.
4. **AI Analysis** (`test.py`) — Sends the latest anomaly data to an OpenAI-based multi-agent system for reasoning, verification, calibration, and reporting.
5. **Scheduling** (`main.py`, `schedu.py`) — Uses the `schedule` library to run the pipeline on a recurring basis (every 1–5 minutes/seconds).
6. **Frontend Dashboard** (`server.py`) — Flask web server serving a Palantir-style dark dashboard that displays catalyst reports, allows running analysis, and shows historical data.

### Frontend Dashboard (`server.py`)

- **Framework**: Flask with Jinja2 templates, static CSS/JS
- **Design**: Dark theme, Palantir-inspired institutional look with Inter + JetBrains Mono fonts
- **Structure**:
  - `templates/dashboard.html` — Main dashboard HTML template
  - `static/css/dashboard.css` — Palantir-style dark theme CSS
  - `static/js/dashboard.js` — Frontend logic for API calls, rendering, interactions

#### API Endpoints:
- `GET /` — Serves the dashboard HTML
- `GET /api/catalysts` — Returns all catalyst reports (limit 50)
- `GET /api/catalysts/latest` — Returns the most recent analysis run
- `GET /api/catalysts/history` — Returns list of past analysis runs
- `GET /api/catalysts/run/<run_id>` — Returns catalysts for a specific run
- `GET /api/stats` — Returns dashboard statistics (counts, averages)
- `POST /api/run-analysis` — Triggers a new AI analysis run (imports from test.py)

#### Catalyst Data Model (CatalystReport from test.py):
- `rank` — Ranking of the catalyst
- `ticker` — Stock ticker (e.g., 1088.HK, CLF, MP)
- `direction` — upside or downside
- `catalyst` — Description of the catalyst event
- `adjusted_impact_usd` — Financial impact in USD
- `confidence_pct` — Confidence level percentage
- `time_horizon` — Expected timeframe (medium-term, long-term)
- `summary` — Concise investor-ready summary

### Key Components

- **`AlphaVantageClient`** (`alpha.py`): Wrapper around the Alpha Vantage REST API.
- **`FXAnomalyDetector`** (`change.py`): Detects anomalies by comparing consecutive daily close prices.
- **`save_event` / `SQL`** (`signal123.py`): PostgreSQL operations for saving and retrieving anomaly events.
- **AI Agent Layer** (`test.py`): Multi-agent system using OpenAI Agents SDK with reason, verification, calibration, and reporting agents.
- **RAG Module** (`rag_main.py`): Retrieval-augmented generation using pgvector for semantic search.
- **Ticker Tools** (`ok.py`): Fetches real-time price and fundamental data from Alpha Vantage and FMP.

### Database Design

PostgreSQL is used with these tables:
- **`signal_11`**: Stores anomaly events
- **`scenario`**: Stores scenario data
- **`china_miningpolicy_scenarios`**: Stores policy scenarios with vector embeddings for RAG
- **`catalyst_reports`**: Stores AI-generated catalyst reports with columns: `id`, `run_id`, `rank`, `ticker`, `direction`, `catalyst`, `adjusted_impact_usd`, `confidence_pct`, `time_horizon`, `summary`, `created_at`

### Scheduling

Two scheduler entry points exist:
- `schedu.py`: Runs anomaly detection and storage every 5 seconds
- `main.py`: Runs the full workflow (data retrieval + AI analysis) every 1 minute

### APIs and Services

- **Alpha Vantage API**: Financial market data. Requires `ALPHA_VANTAGE_API_KEY`.
- **OpenAI API**: AI agent analysis and embeddings. Via `AI_INTEGRATIONS_OPENAI_API_KEY` and `AI_INTEGRATIONS_OPENAI_BASE_URL`.
- **FMP API**: Fundamental financial data. Requires `FMP_API_KEY`.
